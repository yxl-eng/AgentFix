# AgentFix GUI 控制台使用说明

AgentFix Console（GUI 控制台）是为 AgentFix 提供的一个极简、易用的图形化桌面应用。通过它，你可以直观地修改修复配置，并一键启动修复任务，全程无需在终端中敲击复杂的命令。

## 1. 启动方式

在确保你已经安装好 Python 并且处于 AgentFix 根目录下时，直接在终端中运行：

```bash
python gui.py
```

执行后，会弹出一个名为 **AgentFix 控制台** 的窗口。

## 2. 界面结构

控制台界面主要分为**左侧导航栏**和**右侧工作区**。

### 左侧导航栏
左侧导航栏提供两个核心功能入口：
- **配置中心 (Configuration)**：用于管理 `agentfix.yaml` 中的各类配置参数。
- **运行看板 (Dashboard)**：用于选择本地仓库、异常日志并触发自动修复任务。

---

## 3. 功能详解

### 3.1 配置中心 (Configuration)

在这里，你可以对 AgentFix 的各项参数进行可视化修改。界面顶部包含一个 **保存配置 (Save Config)** 按钮，修改完毕后**必须点击该按钮**才能将配置持久化到 `agentfix.yaml` 文件中。

配置项分为 5 个选项卡（Tab）：

1. **系统配置 (System)**
   - **服务主机名 / 端口**：配置 AgentServer 启动时监听的地址和端口。
   - **轮询间隔 / 状态路径**：Webhook 轮询相关配置。
   - **验证用 Python 路径**：Agent 校验补丁时所用的 Python 解释器路径。

2. **模型与推理 (Model & Inference)**
   - **大模型名称 / API 接口地址**：配置你正在使用的 LLM 模型（如 OpenAI、DeepSeek 等）。
   - **推理消耗设置**：调整模型在分析和生成补丁时的推理力度。
   - **安全边界（Guardrails）**：设置最大允许修改的文件数、最大允许补丁行数以及最低置信度。
   - **忽略路径**：可配置逗号分隔的路径（如 `.git, node_modules`），Agent 将在分析时忽略这些目录。

3. **凭据管理 (Credentials)**
   - 统一管理各类敏感 Token 的**环境变量名**（如 `ARK_API_KEY`、`GITHUB_TOKEN` 等）。
   - *注意：这里配置的是读取这些 Token 的环境变量名称，而非明文密码。*

4. **Targets 配置 (Targets)**
   - 展现当前已在 `agentfix.yaml` 中配置的所有目标服务（Targets）列表。
   - *注：复杂结构的目标服务详情编辑即将上线，目前如需修改具体目标服务的测试命令等信息，请直接编辑 `agentfix.yaml`。*

5. **运行与产物 (Runtime)**
   - 配置修复产物（补丁文件、分析结果）和修复记录的存储目录。
   - 配置是否在完成修复后自动提交记录。

### 3.2 运行看板 (Dashboard)

这是执行核心修复流程的工作台。

1. **目标仓库路径 (Repository Address)**
   - 点击 **浏览... (Browse...)** 按钮，选择你要进行修复的本地代码仓库目录（对应命令行的 `--repo`）。

2. **异常日志路径 (Log File Address)**
   - 点击 **浏览... (Browse...)** 按钮，选择包含错误堆栈信息的日志文件（对应命令行的 `--log-file`）。

3. **运行修复**
   - 选好上述两个路径后，点击 **▶ 运行修复 (Run AgentFix)** 按钮。
   - AgentFix 会在后台启动工作流，下方的黑色控制台区域会实时滚动输出 AgentFix 的分析、补丁生成以及验证的详细执行日志。
   - 运行结束后，最后一行会打印 `运行结束，退出码为 X`，退出码为 0 代表修复验证成功。

## 4. 常见问题

- **点击运行没有反应 / 提示错误？**
  请确保你选择了有效的仓库目录和日志文件。
- **配置修改后没有生效？**
  请检查修改后是否点击了右上角的“保存配置 (Save Config)”按钮。
- **输出框乱码或卡住？**
  由于 GUI 是通过子进程调用 CLI，如果修复过程非常耗时，界面可能需要等待片刻才会刷新，请耐心等待运行结束的提示。
