# PatchPilot V4 桌面端控制台 (GUI) 使用指南

PatchPilot 桌面端是一个基于 Python 内置 `tkinter` 开发的本地工作台。它面向普通开发者和运维人员，提供了一个直观的图形界面来管理配置、查看自动修复事故记录、配置目标服务，并支持作为**后台守护进程（Daemon）**常驻系统托盘，实时监控并响应飞书 webhook 或本地日志报错。

---

## 1. 启动与打包

### 1.1 启动 GUI
确保已安装所需的依赖（包括 `pystray`, `pillow`, `pyyaml` 等），在项目根目录运行：

```powershell
python gui.py
```

如果你使用 Conda 环境（例如名为 `patchpilot311`）：
```powershell
conda run --no-capture-output -n patchpilot311 python gui.py
```

### 1.2 打包为可执行文件 (EXE)
如果你希望将其分发为独立的桌面软件，可以使用内置的打包脚本：
```powershell
python build_exe.py
```
这会在 `dist` 目录下生成 `PatchPilot.exe`。

---

## 2. 数据与配置存储

GUI 主要读取和写入以下两个配置文件：
1. `patchpilot.yaml`：项目的默认/基础配置。
2. `patchpilot.local.yaml`：**本地用户配置（推荐）**。
   - 所有的界面保存操作都会将修改写入到 `local.yaml` 中。
   - 本地路径、真实仓库名和敏感密钥（Token、API Key 等）都会存在这里，该文件已在 `.gitignore` 中忽略，不会提交到代码库。

---

## 3. 核心功能模块

### 3.1 守护模式与系统托盘 (Daemon & Tray)
这是 V4 版本引入的重要特性，使 PatchPilot 能够真正在后台提供“监控与自动修复”服务。

- **开启守护模式**：在“手动运行”页面中，勾选 **[守护模式] 关闭窗口后，后台常驻监听 (Agent Serve)**。
- **托盘隐藏**：开启守护模式后，点击窗口右上角的 `X` 关闭按钮，程序**不会退出**，而是会隐藏到 Windows 系统托盘（右下角小图标）。
- **后台执行**：隐藏后，GUI 会在后台启动 `patchpilot serve` 服务，持续监听 webhook 告警并进行自动修复。
- **恢复与退出**：右键点击系统托盘的 PatchPilot 图标，可以选择 **显示面板 (Show)** 恢复界面，或选择 **完全退出 (Quit)** 彻底关闭后台服务和界面。

> **注意**：如果未开启守护模式，点击 `X` 会直接完全退出程序。

### 3.2 总览看板 (Dashboard)
用于快速了解 PatchPilot 的整体运行状况：
- **数据指标**：顶部展示“总事件数”、“已修复 PR 数”、“只报告数”和“已忽略数”的核心 KPI。
- **最近事故**：列表展示最近发生的事件，包含目标服务、状态、处理结论和摘要。
- **一键刷新**：点击右上角“刷新”按钮，可实时同步最新的修复记录。

### 3.3 事故记录 (Incidents)
提供对所有历史报错和修复记录的深度追溯（基于 `records/*.json`）。
- **列表与检索**：左侧列表支持按“事件状态（Status）”筛选，并支持关键字搜索。
- **事故详情**：右侧面板展示详细的分诊结论、风险等级、根因类型。
- **修复过程透明化**：
  - 展示大模型生成的 **修复思路 (Repair Plan)**。
  - 展示 **自动生成测试说明**（包含测试文件路径、覆盖用例等）。
  - 列出所有的 **工具调用链路**（例如读文件、执行测试、Bash 命令等）。
  - 提供指向 GitHub 的 **PR 链接** 和供开发者参考的 **人工处理建议**。

### 3.4 目标服务管理 (Targets)
管理需要被 PatchPilot 监控和自动修复的本地服务。
- **增删改查**：支持“新增”、“复制”和“删除”目标服务，所有服务都在下拉框中切换。
- **核心字段**：
  - `远程仓库`：**（可选）** 用于辅助克隆。如果你本地已经有了代码，直接填下面那行就行。
  - `本地仓库路径`：这个项目在你电脑上的绝对路径。
  - `PR 目标分支`：修复完成后提交 PR 的 Base 分支。
  - `服务日志文件`：**（按需填写）** 仅在你想让 PatchPilot 主动监控本地文件变动（Watch 模式）时填写。如果你打算通过 Webhook 接收线上服务器的报错，则此项留空即可。

### 3.5 配置中心 (Configuration)
集中管理全局变量和敏感密钥。
- **密钥安全**：模型 API Key、GitHub Token、飞书 Webhook 等敏感字段默认使用 `***` 掩码隐藏，可点击“预览”临时查看明文。不会在界面上暴露明文密码。
- **Agent Planner 与风控**：可以调整 Planner 的最大工具步数，以及设定最大修改文件数、最大补丁行数等“护栏（Guardrails）”策略。
- **运行与验证**：配置服务监听端口（默认 8080）、轮询间隔、Python 解释器路径等。

### 3.6 手动运行 (Manual Run)
在不依赖 Webhook 或长期后台监控的情况下，单次触发修复。
- 选择本地仓库路径和对应的日志文件。
- 勾选 **“只验证，不创建 PR”**（默认勾选）可以用于本地安全测试。关闭此项后，修复成功将实际推送分支并创建 PR。
- 点击“开始运行”后，下方的终端黑框会实时打印命令执行日志。

## 4. 典型使用链路 (Workflow)

1. **初始化配置**：打开 GUI，在【配置中心】填好你的大模型 API Key、GitHub Token 和飞书机器人地址，点击保存。
2. **注册目标服务**：进入【目标服务】，新增一个你要监控的本地项目。填好它的本地路径、日志文件路径、验证命令等。
3. **开启后台守护**：进入【手动运行】，勾选“守护模式”。
4. **隐藏至托盘**：点击窗口 `X`，让 PatchPilot 在后台安静运行。
5. **触发报错与自动修复**：当你的目标服务日志中出现异常，或收到外部 Webhook 告警时，后台的 PatchPilot 会自动介入，进行分析、修代码、跑测试。
6. **审查结果**：收到飞书通知后，点击托盘图标恢复界面，进入【事故记录】查看本次修复的 PR 链接、修改思路和人工复核建议。
